Q01. メソッドの役割に関する説明として、最も適切なものはどれですか？
    次の４つから正しい答えを選びなさい。
選択肢：
a. プログラムの実行速度を向上させるための機能。
b. 特定の処理を一つの単位としてまとめ、再利用性を高めるための機能。
c. 変数を宣言するためだけの専用の構文。
d. プログラムのエラーを自動的に修正するための機能。

答え：
b
解説：
メソッドは、関連する一連の処理を名前付きのブロックにカプセル化します。これにより、同じコードを何度も書く必要がなくなり、コードの可読性、保守性、再利用性が向上します。

Q02. メソッドが値を返さないことを示すために使用するキーワードは何ですか？
    次の４つから正しい答えを選びなさい。
選択肢：
a. null
b. none
c. void
d. empty

答え：
c
解説：
メソッドの戻り値の型を`void`と宣言すると、そのメソッドは呼び出し元に値を返さないことを意味します。

Q03. メソッドを定義する際に受け取る値を格納する変数を何と呼びますか？
    次の４つから正しい答えを選びなさい。
選択肢：
a. 引数 (Argument)
b. 仮引数 (Parameter)
c. 戻り値 (Return Value)
d. 変数 (Variable)

答え：
b
解説：
メソッドの定義で宣言される変数を仮引数（パラメータ）と呼びます。一方、メソッドを呼び出すときに渡す実際の値を引数（アーギュメント）と呼びます。

Q04. 同じクラス内に、同じ名前で、異なる型や数の仮引数を持つ複数のメソッドを定義することを何と呼びますか？
    次の４つから正しい答えを選びなさい。
選択肢：
a. オーバーライド (Override)
b. オーバーロード (Overload)
c. 継承 (Inheritance)
d. 再帰 (Recursion)

答え：
b
解説：
オーバーロード（多重定義）により、同じメソッド名で異なるデータ型や引数の数に対応する処理を実装できます。これにより、メソッドの呼び出し側は引数の違いを意識するだけで、同じ名前のメソッドを直感的に使用できます。

Q05. 2つの`int`型の値を受け取り、その合計を返す`add`という名前のメソッドを定義しなさい。
答え：
```java
static int add(int x, int y) {
    return x + y;
}
```
解説：
`int`型の戻り値を指定し、2つの`int`型の仮引数`x`と`y`を宣言します。メソッドの本体では、`x`と`y`の合計を`return`キーワードを使って呼び出し元に返します。

Q06. メソッド自身をそのメソッドの内部から呼び出す手法を何と呼びますか？
    次の４つから正しい答えを選びなさい。
選択肢：
a. 反復 (Iteration)
b. 再帰 (Recursion)
c. 循環 (Circulation)
d. 委譲 (Delegation)

答え：
b
解説：
再帰は、問題をより小さな同じ構造の部分問題に分割して解く場合に有効な手法です。再帰メソッドには、無限ループに陥らないように、処理を終了させるためのベースケース（終了条件）が不可欠です。

Q07. 階乗を計算する再帰メソッド`factorial(int n)`を定義しなさい。階乗とは、1からその数までのすべての正の整数を掛け合わせたものです（例: `5! = 5 * 4 * 3 * 2 * 1`）。なお、`0`の階乗は`1`とします。
答え：
```java
static int factorial(int n) {
    if (n == 0) {
        return 1; // ベースケース
    } else {
        return n * factorial(n - 1); // 再帰呼び出し
    }
}
```
解説：
`n`が`0`の場合に`1`を返すというベースケースを設けます。`n`が`0`より大きい場合、`n`と`factorial(n - 1)`の積を返します。これにより、`n`が1ずつ減少し、最終的に`n=0`のベースケースに到達して計算が完了します。
