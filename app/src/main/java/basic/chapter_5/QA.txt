Q01. Javaの`switch`文に関する説明として、最も適切なものはどれですか？
選択肢：
a. 複雑な条件や範囲指定の分岐に最適である。
b. 一つの変数に対する特定の値での多分岐に得意である。
c. `if-else if-else`文よりも常に高速に動作する。
d. `boolean`型の変数のみを条件として使用できる。
-------------------------------------------------------------------------------

答え：
b
解説：
`switch`文は、一つの変数（または式）の値に基づいて、
複数の処理経路の中から一つを選択する多分岐の制御文です。特定の値との比較に優れています。

Q02. 次のJavaコードが実行されたとき、出力される結果として正しいものはどれですか？
```java
int day = 3;
String dayName;
switch (day) {
    case 1:
        dayName = "月曜日";
    case 2:
        dayName = "火曜日";
    case 3:
        dayName = "水曜日";
    default:
        dayName = "不明";
}
System.out.println(dayName);
```
選択肢：
a. 月曜日
b. 水曜日
c. 不明
d. コンパイルエラー
-------------------------------------------------------------------------------

答え：
c
解説：
`case 3`に合致した後、`break`文がないため、後続の`case`や`default`の処理が
実行されます（フォールスルー）。結果として`dayName`には`"不明"`が代入されます。

-------------------------------------------------------------------------------
Q03. `switch`文において、どの`case`にも合致しない場合の処理を記述するために
     使用するキーワードは何ですか？
選択肢：
a. `else`
b. `finally`
c. `default`
d. `otherwise`
-------------------------------------------------------------------------------

答え：
c
解説：
`default`句は、`switch`文の式がどの`case`ラベルの値とも一致しなかった場合に
実行される処理を定義します。

Q04. Java 14以降で導入された新しい`switch`式（switch expressions）の主な利点は何ですか？
選択肢：
a. `break`文を完全に不要にする。
b. `switch`文を式として使用し、値を返すことができる。
c. `switch`文内で`if`文を使用できなくなる。
d. `switch`文で`boolean`型の値を直接扱えるようになる。
-------------------------------------------------------------------------------

答え：
b
解説：
新しい`switch`式は、`switch`文を式として扱うことができ、結果として値を返すことができます。
これにより、コードがより簡潔になり、フォールスルーによるバグのリスクも軽減されます。

Q05. ユーザーから入力された数値`num`が、`1`から`5`の範囲内であれば「小」、
`6`から`10`の範囲内であれば「中」、それ以外であれば「大」と表示するプログラムを記述する場合、
`if`文と`switch`文のどちらがより適切ですか？また、その理由も簡潔に述べなさい。
-------------------------------------------------------------------------------

答え：
`if`文がより適切です。
解説：
`switch`文は特定の値での多分岐に得意ですが、範囲指定のような複雑な条件には対応できません。
このような場合は、条件式で範囲を表現できる`if`文（`if (num >= 1 && num <= 5)`など）を
使用する方が適切で、コードの可読性も高まります。
