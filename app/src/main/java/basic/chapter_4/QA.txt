Q01. プログラムの流れをコントロールする仕組みである制御文の基本型として
     正しい組み合わせはどれですか？
選択肢：
a. 順次、分岐、並列
b. 分岐、繰り返し、並列
c. 順次、分岐、繰り返し
d. 順次、並列、繰り返し
-------------------------------------------------------------------------------

答え：
c
解説：
制御文の基本型は、順次処理、条件による分岐処理、そして繰り返し処理の3つです。

Q02. 次のJavaコードが実行されたとき、出力される結果として正しいものはどれですか？
```java
int x = 10;
if (x > 5) {
    System.out.println("xは5より大きい");
}
```
選択肢：
a. xは5より大きい
b. 何も出力されない
c. コンパイルエラー
d. 実行時エラー
-------------------------------------------------------------------------------

答え：
a
解説：
`if (x > 5)`の条件式は`true`となるため、中括弧`{}`内の
`System.out.println("xは5より大きい");`が実行されます。

```java
int score = 75;
if (score >= 80) {
```

a. 合格
b. 不合格
c. コンパイルエラー
d. 実行時エラー
-------------------------------------------------------------------------------

答え：
b
解説：
`score >= 80`の条件式は`false`となるため、`else`ブロック内の`System.out.println("不合
格");`が実行されます。

Q04. 次のJavaコードが実行されたとき、出力される結果として正しいものはどれですか？
```java
int temperature = 25;
if (temperature > 30) {
    System.out.println("暑い");
} else if (temperature > 20) {
    System.out.println("快適");
} else {
    System.out.println("寒い");
}
```
選択肢：
a. 暑い
b. 快適
c. 寒い
d. 何も出力されない
-------------------------------------------------------------------------------

答え：
b
解説：
`temperature > 30`は`false`ですが、次の`temperature > 20`は`true`となるため、
そのブロック内の`System.out.println("快適");`が実行されます。
`if-else if-else`の連鎖では、最初に`true`になったブロックだけが実行されます。

Q05. Javaの`if`文において、処理が1行であっても中括弧`{}`を付けることが
     推奨される主な理由は何ですか？
選択肢：
a. コンパイルエラーを避けるため
b. 実行速度を向上させるため
c. コードの可読性を高め、将来の変更時のバグを防ぐため
d. メモリ使用量を削減するため
-------------------------------------------------------------------------------

答え：
c
解説：
処理が1行の場合でも中括弧`{}`を付けることで、コードの範囲が明確になり、可読性が向上します。
また、後から処理を追加する際に中括弧の付け忘れによる意図しないバグ
（例えば、追加した行が`if`文の制御下にないなど）を防ぐことができます。

Q06. あるECサイトのバーゲンセールで購入金額合計が変数名totalに入っていて、
     1000円以上は10%引き、5000円以上で20%引になる場合、
     小数型変数名diccountに割引率を代入する式を三項演算子で答えなさい（例：10%の時は0.1）。
-------------------------------------------------------------------------------

答え：
diccount = total>=5000 ? 0.2 : (total>=1000 ? 0.1 : 0);
解説：
最初どの合計金額と比べるか順番を注意しないといけません。
