Q01. プログラムのソースコードにおいて、数値を直接記述したものを何と呼びますか？
    次の４つから正しい答えを選びなさい。
-------------------------------------------------------------------------------
選択肢：
a. 変数
b. リテラル
c. 式
d. 演算子

答え：
b
解説：
リテラルは、ソースコード内で直接表現される値（数値、文字列など）です。

Q02. Javaの基本データ型（プリミティブ型）に含まれないものはどれですか？
    次の４つから正しい答えを選びなさい。
選択肢：
a. int
b. double
c. String
d. boolean
-------------------------------------------------------------------------------

答え：
c
解説：
`String`は、クラスとして定義されている参照型です。
`int`, `double`, `boolean`は基本データ型です。

-------------------------------------------------------------------------------
Q03. 3.14という浮動小数点数のリテラルを`float`型の変数に代入する正しい記述はどれですか？
    次の４つから正しい答えを選びなさい。
選択肢：
a. float f = 3.14;
b. float f = 3.14d;
c. float f = 3.14f;
d. float f = (float)3.14L;
-------------------------------------------------------------------------------

答え：
c
解説：
Javaでは、小数点を含むリテラルはデフォルトで`double`型として扱われます。
`float`型の変数に代入するには、リテラルの末尾に`f`または`F`を付けて`float`型であることを
明示する必要があります。

Q04. 一度値を代入したら変更できない変数を宣言するために使用するキーワードは何ですか？
    次の４つから正しい答えを選びなさい。
選択肢：
a. static
b. final
c. const
d. fixed
-------------------------------------------------------------------------------

答え：
b
解説：
`final`キーワードを付けて宣言された変数は定数となり、再代入が禁止されます。

Q05. 文字列の中にダブルクォーテーション（"）を含めるために使用する
エスケープシーケンスは何ですか？  次の４つから正しい答えを選びなさい。
選択肢：
a. \'
b. \"
c. \\
d. \n
-------------------------------------------------------------------------------

答え：
b
解説：
`\"`と記述することで、文字列リテラル内にダブルクォーテーション文字そのものを含めることができます。

Q06. 5つの整数を格納できる配列を宣言し、初期化するコードを記述しなさい。
-------------------------------------------------------------------------------

答え：
```java
// 方法1: 宣言と同時に初期化
int[] numbers = {10, 20, 30, 40, 50};

// 方法2: サイズを指定して配列オブジェクトを生成
int[] numbers2 = new int[5];
```
解説：
配列は`型[] 配列名`で宣言します。`{}`を使って初期値を指定する方法と、
`new`キーワードを使って指定したサイズの配列オブジェクトを生成する方法があります。

Q07. 整数型の変数`a`を宣言し、値`10`を代入し、
その値をコンソールに出力するプログラムを作成しなさい。
-------------------------------------------------------------------------------

答え：
```java
public class StatementExample {
    public static void main(String[] args) {
        int a;
        a = 10;
        System.out.println(a);
    }
}
```
解説：
`int a;`で整数型の変数を宣言し、`a = 10;`で値を代入します。
`System.out.println(a);`で変数の値を出力します。
宣言と代入は`int a = 10;`のように一行で書くこともできます。

Q08. `int`型の配列`myArray`の要素数が10であるとき、
`myArray[10]`にアクセスしようとするとどうなりますか？  次の４つから正しい答えを選びなさい。
選択肢：
a. `0`が返される
b. `null`が返される
c. コンパイルエラーが発生する
d. `ArrayIndexOutOfBoundsException`が発生する
-------------------------------------------------------------------------------

答え：
d
解説：
配列のインデックスは`0`から`要素数-1`までです。
要素数が10の場合、有効なインデックスは0から9までとなり、`10`は範囲外であるため、
実行時に`ArrayIndexOutOfBoundsException`という例外が発生します。

Q01. Javaの識別子（変数名やメソッド名など）として使用できないものはどれですか？
    次の４つから正しい答えを選びなさい。
選択肢：
a. _myVariable
b. $myVariable
c. 1stVariable
d. myVariable1
-------------------------------------------------------------------------------

答え：
c
解説：
識別子の先頭に数字を使うことはできません。識別子は、文字、アンダースコア(_)、
ドル記号($)で始める必要があります。

-------------------------------------------------------------------------------
Q02. Javaの識別子に関する説明として間違っているものはどれですか？
    次の４つから正しい答えを選びなさい。
選択肢：
a. 大文字と小文字は区別される。
b. 予約語（キーワード）を使用することができる。
c. 長さに制限はない。
d. 使える記号は`_`と`$`のみである。
-------------------------------------------------------------------------------

答え：
b
解説：
`class`や`int`などの予約語（キーワード）は、識別子として使用することはできません。

-------------------------------------------------------------------------------
Q03. Javaのコーディング規約において、変数名の命名で一般的に推奨されている記法はどれですか？
    次の４つから正しい答えを選びなさい。
選択肢：
a. スネークケース (my_variable)
b. キャメルケース (myVariable)
c. パスカルケース (MyVariable)
d. ケバブケース (my-variable)
-------------------------------------------------------------------------------

答え：
b
解説：
Javaでは、クラス名はパスカルケース（アッパーキャメルケース）、
変数名とメソッド名はキャメルケース（ローワーキャメルケース）で記述するのが一般的です。

-------------------------------------------------------------------------------
Q04. Javaの定数（`final`修飾子が付いた変数）の命名で一般的に推奨されている記法はどれですか？
  次の４つから正しい答えを選びなさい。
選択肢：
a. すべて小文字のスネークケース (max_value)
b. キャメルケース (maxValue)
c. すべて大文字のスネークケース (MAX_VALUE)
d. パスカルケース (MaxValue)
-------------------------------------------------------------------------------

答え：
c
解説：
定数は、すべての文字を大文字にし、単語間をアンダースコアで区切るスネークケースで
記述するのが一般的です。これにより、通常の変数と区別しやすくなります。

Q05. `userName`と`username`という2つの変数を宣言した場合、
    これらは同じ変数として扱われますか？
-------------------------------------------------------------------------------

答え：
いいえ、扱われません。Javaは大文字と小文字を区別するため、
`userName`と`username`は異なる変数として認識されます。
解説：
Javaの識別子はケースセンシティブです。したがって、大文字と小文字の違いは厳密に区別されます。
